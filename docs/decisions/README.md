# Decisions

This folder stores lightweight architecture and product decisions for the repository.

Use a decision note when:
- a choice affects multiple tasks
- the repo should stay consistent over time
- we want to record rejected alternatives and consequences

Use `docs/tasks/` for implementation contracts.
Use `docs/decisions/` for durable decisions about how and why the system works.
